<h2 id="What-is-Python?">What is Python?<a class="anchor-link" href="#What-is-Python?">&#182;</a></h2><p>Python is now 26 years old. Its creator, <a href="https://en.wikipedia.org/wiki/Guido_van_Rossum">Guido van Rossum</a>, named it after the British comedy "Monty Python's Flying Circus." His goals for the language were that it be an "an easy and intuitive language just as powerful as major competitors," producing computer code "that is as understandable as plain English."</p>

<p>It is a general-purpose language, which means that you can use it for anything:  organizing data, scraping the web, creating websites, analyzing sounds, creating games, and of course <em>engineering computations</em>.</p>

<p>Python is an interpreted language. This means that you can write Python commands and the computer can execute those instructions directly. Other programming languages—like C, C++ and Fortran—require a previous <em>compilation</em> step: translating the commands into machine language.

A neat ability of Python is to be used <em>interactively</em>. <a href="https://en.wikipedia.org/wiki/Fernando_Pérez_(software_developer">Fernando Perez</a>) famously created <strong>IPython</strong> as a side-project during his PhD. We're going to use IPython (the I stands for "interactive") in this lesson.</p>

You can find lots of resources online and, since Python is an open-source project, you'll also find a friendly community of people sharing their knowledge.</p>

<p>Python is known as a <em>high-productivity language</em>. As a programmer, you'll need less time to develop a solution with Python than with most languages. 

This is important to always bring up whenever someone complains that "Python is slow."

Your time is more valuable than a machine's!

Because Python plays well with other languages :–)</p>

<h5 id="Key-concept:-function">Key concept: function<a class="anchor-link" href="#Key-concept:-function">&#182;</a></h5><p>A function is a compact collection of code that executes some action on its <em>arguments</em>.  Every Python function has a <em>name</em>, used to call it, and takes its arguments inside round brackets. Some arguments may be optional (which means they have a default value defined inside the function), others are required. For example, the <code>print()</code> function has one required argument: the string of characters it should print out for you.</p>

<p>Python comes with many <em>built-in</em> functions, but you can also build your own. Chunking blocks of code into functions is one of the best strategies to deal with complex programs. It makes you more efficient, because you can reuse the code that you wrote into a function. Modularity and reuse are every programmer's friend.</p>

<h3 id="Variables-and-their-type">Variables and their type<a class="anchor-link" href="#Variables-and-their-type">&#182;</a></h3><p>Variables consist of two parts: a name and a value. When we want to give a variable its name and value, we use the equal sign: <code>name = value</code>. This is called an <em>assignment</em>. The name of the variable goes on the left and the value on the right.</p>

<p>The first thing to get used to is that the equal sign in an assignment has a different meaning than it has in Algebra! Think of it as an arrow pointing from <code>name</code> to <code>value</code>.</p>

<p><img src="images/variables.png" style="width: 400px;"/></p>

<p>We have many possibilities for variable names: they can be made up of upper and lowercase letters, underscores and digits… although digits cannot go on the front of the name. For example, valid variable names are:</p>

<div class="highlight"><pre><span class="n">x</span>

    <span class="n">x1</span>

    <span class="n">X_2</span>

    <span class="n">name_3</span>

    <span class="n">NameLastname</span>

</pre></div>

<p>Keep in mind, there are reserved words that you can't use; they are the special Python <a href="https://docs.python.org/3/reference/lexical_analysis.html#keywords">keywords</a>.</p>

<p>Logical operators are the following: <code>and</code>, <code>or</code>, and <code>not</code>. They work just like English (with the added bonus of being always consistent, not like English speakers!).  A logical expression with <code>and</code> is <code>True</code> if both operands are true, and one with <code>or</code> is <code>True</code> when either operand is true. And the keyword <code>not</code> always negates the expression that follows.</p>

<p>Let's do some examples:</p>
